Overview

Project Summary: This Typescript project features a login page and a user dashboard, designed with a focus on responsiveness and user experience. The app fetches user data from a mock API and allows for dynamic filtering between users. Styling was implemented using SCSS/Sass for modular and maintainable code.
Role: Web Developer

Tech Stack & Tools

List of Technologies Used:

  • Frontend: TypeScript, React.js
  • Styling: SCSS/Sass
  • Data Handling: Mock API, Fetch/Axios
  • State Management: React Hooks (useState, useEffect)

    • Login Page

    Implementation:

  • Developed a secure and responsive login page using TypeScript and React.js.
  • Managed login state using useState
  • Handled form validation and user feedback for login errors.

    • Challenges & Solutions:

  • Challenge: Ensuring form validation across different inputs.
  • Solution: Used controlled inputs and TypeScript type definitions to validate form data.

    • User Dashboard

    Implementation:

  • Created a structured and responsive user dashboard.
  • Fetched user details from a mock API using Axios and managed state using useEffect.
  • Enabled real-time filtering between users based on selected criteria.

    • Filter Function:

  • Implemented a search bar with dynamic filtering.
  • Used string matching and array filtering to update the list in real time.

    • Challenges & Solutions:

  • Challenge: Ensuring the filter updates state efficiently without lag.
  • Solution: Used debouncing to reduce state updates and improve performance.

    • Styling with SCSS/Sass

    Implementation:

  • Used SCSS/Sass for modular and reusable styles.
  • Created separate style files for components to maintain clean code structure.
  • Utilized variables and mixins for consistent styling and easy updates.

    • Example:

  • Defined color themes and responsive breakpoints using SCSS variables.
  • Used mixins for consistent button styling and hover states.

    • Challenges & Solutions:

  • Challenge: Managing style consistently across components.
  • Solution: Created global style variables and mixins for easy reuse.

    • Responsive Design

    Implementation:

  • Ensured responsiveness using CSS media queries and flexbox/grid.
  • Adapted layout to different screen sizes (mobile, tablet, desktop).

    • Example:

  • Adjusted navbar and dashboard layout for mobile view.
  • Used flexbox for consistent alignment and spacing.

    • Challenges & Solutions:

  • Challenge: Maintaining consistent layout across different resolutions.
  • Solution: Used flexbox and grid for adaptive resizing and positioning.
